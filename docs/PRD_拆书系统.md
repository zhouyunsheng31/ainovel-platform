## 一、产品概述

### 1.1 产品定位
面向小B端（小型企业/工作室/个人作者）的AI小说分析与拆解平台，通过AI技术自动化提取小说的结构化信息，生成多层次的"纲"（章纲→粗纲→大纲→世界纲），帮助用户快速理解小说的创作套路和结构。

### 1.2 目标用户
- 网文工作室编辑
- 小说作者（学习优秀作品结构）
- 内容分析师
- IP运营人员

### 1.3 核心价值
1. **自动化拆解**：一键上传小说，AI自动完成多层级结构提取
2. **结构化输出**：生成可复用的写作模板和知识图谱
3. **可视化呈现**：以组织架构图形式展示所有"纲"的层级关系
4. **进度可监控**：实时查看每个处理步骤的状态和错误信息

## 三、功能详细需求

### 3.1 文件上传模块

#### 3.1.1 支持格式
- TXT（UTF-8/GBK/GB2312自动检测）
- EPUB
- DOC/DOCX
- PDF

#### 3.1.2 上传限制
- 单文件最大：50MB
- 文本长度：无硬性限制（分批处理）

#### 3.1.3 解码检测逻辑
1. 优先尝试 UTF-8
2. 失败则尝试 GBK
3. 失败则尝试 GB2312
4. 均失败则提示用户指定编码

### 3.2 文本预处理模块

#### 3.2.1 标题删除规则
删除以下模式的标题行：
- `第X章` / `第X节` / `Chapter X`
- `第一章` / `第二章` ... `第九十九章`
- 纯数字标题行

#### 3.2.2 智能分割规则
- **目标长度**：每章约2000字
- **完整性约束**：
  - 不切断句子（以句号、感叹号、问号结尾）
  - 不切断段落（以换行符为界）
- **分割策略**：
  1. 按2000字初步定位分割点
  2. 向后查找最近的段落结束位置
  3. 在段落边界处分割

### 3.3 章纲提取模块

#### 3.3.1 处理流程
```
原文章节 → LLM（章纲提取提示词） → JSON结构化输出
```

#### 3.3.2 LLM配置
| 参数 | 值 |
|-----|-----|
| Base URL | https://api.n1n.ai |
| API Key | <YOUR_API_KEY> |
| Model | gpt-5.4-nano |

#### 3.3.3 提示词要求
- 使用预定义的"章纲提取"提示词模板
- 剧情概括约束为200字左右
- 输出为JSON格式

#### 3.3.4 并行处理
- 所有章节并行发送LLM请求
- 每个章节独立处理，互不依赖
- 失败章节记录错误，不影响其他章节

### 3.4 粗纲生成模块

#### 3.4.1 输入来源
- 取10份章纲的"概括"字段
- 合并约2000字内容

#### 3.4.2 提示词修改规则
基于"章纲提取"提示词修改：
1. 标题：章纲 → 粗纲
2. 删除："通用型微观修辞"部分
3. 修改："全局一般节奏"改为"剧情节奏分析"
4. 删除：所有"文风提取"（plot_elements下的虚/实分析）
5. 新增：每份章纲标注"起承转合"节拍

#### 3.4.3 处理逻辑
- 每10份章纲生成1份粗纲
- 不足10份的最后一组单独处理

### 3.5 大纲生成模块

#### 3.5.1 输入来源
- 取10份粗纲的"概括"字段

#### 3.5.2 提示词修改规则
- 基于粗纲提示词
- 标题：粗纲 → 大纲

### 3.6 世界纲生成模块

#### 3.6.1 输入来源
- 取所有大纲的"概括"字段

#### 3.6.2 提示词修改规则
- 基于大纲提示词
- 标题：大纲 → 世界纲
- 提取：世界观、人物关系、核心设定等

### 3.7 进度监控模块

#### 3.7.1 监控维度
| 阶段 | 状态 | 进度 |
|-----|------|-----|
| 文件上传 | 等待/进行中/完成/失败 | - |
| 文本预处理 | 等待/进行中/完成/失败 | 百分比 |
| 章纲提取 | 等待/进行中/完成/失败 | X/总数 |
| 粗纲生成 | 等待/进行中/完成/失败 | X/总数 |
| 大纲生成 | 等待/进行中/完成/失败 | X/总数 |
| 世界纲生成 | 等待/进行中/完成/失败 | - |

#### 3.7.2 错误记录
- 记录每个失败步骤的错误信息
- 支持查看详细错误堆栈
- 标记失败的具体章节/位置

### 3.8 纲展示模块

#### 3.8.1 展示形式
- **层级图表**：类似组织架构图
  ```
  世界纲
    └── 大纲1
          ├── 粗纲1-1
          │     ├── 章纲1-1-1
          │     ├── 章纲1-1-2
          │     └── ...
          ├── 粗纲1-2
          └── ...
    └── 大纲2
          └── ...
  ```

#### 3.8.2 交互功能
- 点击节点：展开/收起详情
- 详情面板：显示完整内容
- 复制按钮：一键复制当前纲的全部内容
- 连线追踪：可顺着连线找到上下级关系

#### 3.8.3 实时更新
- 支持展示正在处理中的书籍
- 新生成的纲实时添加到图表
- 处理中的节点显示加载状态

## 四、非功能需求

### 4.1 响应式设计
- 支持移动端（375px+）
- 支持桌面端（1024px+）
- 自适应布局

### 4.2 性能要求
- 文件上传响应时间 < 3秒
- 单章节LLM处理超时：60秒
- 支持100+章节并行处理

### 4.3 可用性
- 界面简洁直观
- 关键操作有确认提示
- 错误信息友好易懂

## 五、技术约束

### 5.1 AI技术栈（强制）
- **LangChain**: LLM调用封装
- **LangGraph**: 工作流编排与状态管理

- 纯HTML/CSS/JavaScript（无框架要求）
- 响应式布局
- 图表可视化

- Python + FastAPI（推荐）
